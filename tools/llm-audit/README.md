# llm-audit — LLM 应用安全查漏工具

基于 L1–L8 分层攻击面的 **轻量级自查工具**，帮助你在新设计/review 一个 LLM 应用时快速定位安全缺口。

## 设计理念

1. **知识库驱动**：所有检查点都在 `checklist.yaml` 里，易于扩展和审阅。
2. **零依赖**：仅 Python 3.8+ 标准库，`python audit.py` 即可运行。
3. **分层评分**：按 L1–L8 分别打分，突出高风险层。
4. **面向整改**：报告不是"pass/fail"，而是按权重排序的**具体建议**。
5. **CI 友好**：支持 `--answers file.json` 非交互模式 + JSON 输出。

## 快速使用

```bash
# 交互式自查（推荐首次使用）
python audit.py

# 只查某一层
python audit.py --layer L3

# 仅打印清单（不回答）
python audit.py --non-interactive

# 从 JSON 读取答案，生成 Markdown 报告
python audit.py --answers my-answers.json --output report.md
```

### 答案文件格式（`my-answers.json`）

```json
{
  "L1-001": "yes",
  "L1-002": "no",
  "L3-001": "yes",
  "L6-001": "na"
}
```

回答取值：
- `yes` — 已满足
- `no`  — 未满足（会进入缺口清单）
- `na`  — 不适用（不计入分母）
- `skip`— 未判断（计入未达标，需人工复核）

## 清单结构

```yaml
layers:
  - id: L1
    name: 输入层 (Input)
    items:
      - id: L1-001
        title: 输入 Unicode 归一化
        question: 应用是否对输入做统一的 Unicode 归一化？
        weight: 2            # 1=一般 2=重要 3=关键
        rationale: ...       # 为什么重要（根因）
        mitigation: ...      # 未满足时的建议动作
```

## 评分规则

- 每层：`得分 = Σ(满足项权重) / Σ(全部项权重)` （跳过 `na`）
- 等级：A(>=90%) / B(>=75%) / C(>=60%) / D(>=40%) / F(<40%)
- 缺口清单：所有回答 `no/skip` 的项，按权重倒序列出，带根因与整改建议

## 扩展清单

直接编辑 `checklist.yaml` 即可。新增条目时遵守：

- `id` 格式 `Lx-NNN`，x 为层号，NNN 三位数字
- 必填字段：`id / title / question / weight / rationale / mitigation`
- `weight`：1=一般, 2=重要, 3=关键（关键项在缺口清单里优先展示）

## 局限

- 本工具是**清单法自查**，不做动态攻击测试。
- 权重和清单基于当前（2026-05）公开知识与本仓库分析，需要随攻击面演化持续维护。
- 不替代专业红队评估。
